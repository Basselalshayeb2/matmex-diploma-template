\section{Заключение}
\label{sec:conclusion}

В рамках практики была разработана и исследована подсистема голосового управления для МИС «СТОММИС» в части заполнения зубной формулы и результатов первичного осмотра пациентов, ориентированная на использование голосовых команд в процессе лечения без необходимости постоянного взаимодействия с интерфейсом.
Ключевая идея решения --- многоэтапная обработка аудио: лёгкая сегментация речи в браузере (VAD), что снижает вычислительную и стоимостную нагрузку по сравнению с непрерывным распознаванием всего потока.

В ходе работы также рассматривались альтернативные способы активации голосового режима, включая ручную активацию (\emph{push-to-talk}) и активацию по ключевому слову (\emph{wake word}).
Практические ограничения данных подходов послужили основанием для перехода к автоматическому выделению команд на основе классификации транскрипции.

Основные результаты работы, соответствующие поставленным задачам:
\begin{itemize}
    \item выполнен обзор существующих решений в области голосового управления, выделения речевых сегментов, идентификации команд и интерпретации распознанного текста; по результатам обзора обоснован выбор архитектуры с VAD в браузере, ASR на сервере и фильтрацией команд по транскрипции;
    \item спроектирована архитектура подсистемы голосового управления для МИС «СТОММИС»: определены основные компоненты, границы ответственности, основной сценарий взаимодействия и место модулей интеллектуальной обработки команд в общем конвейере;
    \item реализован клиентский модуль захвата аудио и детекции речи в браузере с использованием Web Audio API и VAD на базе \texttt{@ricky0123/vad-web} / ONNX Runtime Web;
    \item реализована логика выделения речевого сегмента: правило окончания по паузе (например, 2~секунды тишины) и передача сегмента на сервер в формате \texttt{multipart/form-data} с полем \texttt{file};
    \item реализован серверный конвейер обработки: приём \texttt{file}, транскодирование аудио (FFmpeg $\rightarrow$ Ogg Opus), распознавание речи через Yandex SpeechKit, классификация транскрипции с помощью бинарного классификатора \textit{command/not-command}, интерпретация команды и применение результата к карте через REST API МИС «СТОММИС»;
    \item реализован безопасный слой интерпретации команд: использование LLM \texttt{qwen3-235b-a22b-fp8} с ограничением вывода (JSON-контракт, whitelist интентов/слотов, валидация, режим \texttt{needs\_clarification}); проведён предварительный анализ альтернатив, включая \texttt{gpt-oss-20b};
    \item проведена экспериментальная оценка и пилотная апробация в СПб ГБУЗ «СП №8»; получены предварительные результаты по задержке, снижению объёма обрабатываемого аудио и качеству классификации команд, при этом требуется дальнейшая стабилизация в условиях шума и вариативной речи врачей.
\end{itemize}

Направления дальнейшей работы включают расширение набора команд, улучшение устойчивости к шумам, а также уточнение механизма ограничения допустимых действий при интерпретации команд (в частности, формализацию схемы ответа и справочников предметной области).
Полученные результаты подтверждают возможность практического применения предложенного подхода для автоматизации работы врача в медицинских информационных системах с использованием методов машинного обучения и языковых моделей
без усложнения пользовательского взаимодействия.
